## Overview
Levels 1, 2, 3 were not necessary to solve the challenge. The core exploit was in Admin, which involved forging an
admin JWT using RS256 → HS256 algorithm confusion.

PortSwigger sig2n tool:

The tool computed candidate RSA moduli (n) at multipliers 1, 2, 3, and 6. Each candidate produced a Base64-
encoded X.509 public key and a tampered HS256-signed JWT.
Step 2 — Identify the correct key
Each tampered JWT was tested as the session cookie against /dashboard. The multiplier 1 X.509 key returned 200
OK, confirming it as the real server public key.

Step 3 — Build the malicious signing key in Burp
In Burp's JWT Editor Keys tab, a new Symmetric Key was created. The generated k value was replaced with the
Base64-encoded X.509 public key from Step 5. This sets up Burp to use the server's own RSA public key as an HS256
HMAC secret.

Step 4 — Forge and sign the admin token
In Burp Repeater, the JWT was modified: alg set to HS256, role changed to admin. The token was signed with the
symmetric key from Step 6 using the "Don't modify header" option.


Step 5 — Get the flag
Sending the forged token to /admin/flag returned the Admin Control Panel with the flag:

recon{t00K_n0_1oNS_T0_pUII__!}
